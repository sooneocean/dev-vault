## Friction

<!-- 今天遇到的摩擦點、手動重複做的事、缺少的工具或指引。/improve 會讀取此段落。 -->
- obsidian-agent CLI 的 `updateNote()` 只能更新已存在的 key，無法 INSERT 新 frontmatter field — 遷移必須用 direct file editing
- obsidian-agent CLI 的 template engine 只替換 5 個 known keys（TITLE/DATE/TYPE/GOAL/CONTENT），新欄位的 {{PLACEHOLDER}} 會殘留
- `obsidian-agent journal` 用 UTC 時區判斷日期，跨午夜後仍產出前一天的 journal
- relation_map 沒有任何 CLI consumer — 語義資訊只活在 frontmatter 中，_graph.md 不反映

## Ideas

- obsidian-agent CLI 可以支援 `--fields` flag 讓 `updateNote()` 插入新 key
- cluster 演算法可以考慮 subtype/domain 作為分群維度，不只靠 tag + related
- relation_map 可以被未來的 MCP tool 或 slash command 消費（例如 /graph --typed）
