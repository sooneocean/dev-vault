# php-blog-mcp-server 使用說明

## 這是什麼？

一個 MCP (Model Context Protocol) Server，讓 AI 助手（如 Claude）能直接操作你的 PHP 部落格網站（WordPress）。

```
你 (對話) → Claude (AI) → MCP Server → WordPress REST API → 你的網站
```

你只需要用自然語言下指令，Claude 就能幫你：建立文章、分析 SEO、管理分類標籤、生成 sitemap 等。

---

## 前置需求

| 項目 | 要求 |
|------|------|
| Node.js | ≥ 18 |
| pnpm | 已全域安裝 |
| WordPress 網站 | 需啟用 REST API（預設開啟） |
| MCP Client | Claude Desktop / Cursor / 其他支援 MCP 的工具 |

---

## Step 1: WordPress 產生 Application Password

WordPress 需要「應用程式密碼」來讓 MCP Server 連接（不是你的登入密碼）。

1. 登入 WordPress 後台
2. 進入 `使用者 → 個人資料`
3. 滾到最下面找到 **「應用程式密碼」** 區塊
4. 輸入名稱（例如 `mcp-server`），點「產生新的應用程式密碼」
5. **複製產生的密碼**（格式：`xxxx xxxx xxxx xxxx xxxx xxxx`）

> ⚠️ 這組密碼只會顯示一次，請先存好。

---

## Step 2: 設定環境變數

```bash
# 在項目根目錄
cp .env.example .env
```

編輯 `.env`：

```env
# 你的 WordPress 網址（結尾不要加 /）
WP_BASE_URL=https://your-blog.com

# WordPress 使用者名稱（管理員帳號）
WP_USERNAME=admin

# Step 1 產生的應用程式密碼
WP_APP_PASSWORD=xxxx xxxx xxxx xxxx xxxx xxxx

# 以下用預設值即可
LOG_LEVEL=info
MAX_RETRIES=3
REQUEST_TIMEOUT_MS=10000
DEFAULT_LANGUAGE=zh-TW
SITEMAP_PING_GOOGLE=false
```

---

## Step 3: 安裝並啟動

```bash
# 安裝依賴
pnpm install

# 編譯 TypeScript
pnpm run build

# 啟動 MCP Server（stdio 模式）
pnpm start
```

正常啟動會看到：
```json
{"timestamp":"...","level":"info","message":"Starting php-blog-mcp-server"}
{"timestamp":"...","level":"info","message":"Server connected and ready"}
```

> MCP Server 使用 stdio 傳輸，不需要監聽 port。它由 MCP Client 自動啟動。

---

## Step 4: 連接 MCP Client

### Claude Desktop

編輯設定檔：

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "php-blog": {
      "command": "node",
      "args": ["C:/path/to/php-blog-mcp-server/dist/index.js"],
      "env": {
        "WP_BASE_URL": "https://your-blog.com",
        "WP_USERNAME": "admin",
        "WP_APP_PASSWORD": "xxxx xxxx xxxx xxxx xxxx xxxx"
      }
    }
  }
}
```

重啟 Claude Desktop，就會看到 `php-blog` 工具可用。

### Cursor / 其他 MCP Client

參考各 client 的 MCP 設定方式，指向：
- **Command**: `node`
- **Args**: `["dist/index.js"]`
- **Working directory**: 項目根目錄

## SEO 自動化工作流範例

### 範例 1：一鍵優化文章

對 Claude 說：

> 「幫我分析文章 ID 42 的 SEO 狀況，然後自動生成更好的 meta tags」

Claude 會依序：
1. 呼叫 `content_get_post` 取得文章
2. 呼叫 `seo_analyze_content` 分析評分
3. 呼叫 `seo_generate_meta` 生成優化後的 meta
4. 回傳結果給你

### 範例 2：批次 SEO 健檢

> 「幫我掃描網站前 50 篇文章，找出 SEO 有問題的文章」

Claude 呼叫 `seo_health_check`，回傳所有有問題的文章列表。

### 範例 3：自動發文

> 「幫我寫一篇 800 字關於 2026 年 AI 趨勢的文章，加上適當的 SEO meta，設為草稿」

Claude 會：
1. 生成文章內容
2. 呼叫 `seo_generate_meta` 生成 meta
3. 呼叫 `content_create_post` 建立草稿

## 疑難排解

### 401 Unauthorized
- 確認 `.env` 中的 `WP_USERNAME` 和 `WP_APP_PASSWORD` 正確
- 確認 Application Password 是用管理員帳號產生的
- 確認 WordPress 網站沒有封鎖 REST API

### 403 Forbidden
- 帳號權限不足，需管理員或編輯權限

### 連線逾時
- 檢查 `WP_BASE_URL` 是否正確（不要加尾端 `/`）
- 確認網站有 HTTPS（Application Password 需要安全連線）
- 嘗試增加 `REQUEST_TIMEOUT_MS`

### Tools 沒有出現在 Claude
- 確認 MCP Server 路徑正確
- 檢查 `claude_desktop_config.json` 格式是否正確（JSON 不要有尾逗號）
- 重啟 Claude Desktop

### 查看日誌
```bash
# 設定 debug 模式查看詳細日誌
LOG_LEVEL=debug pnpm start
```
日誌輸出在 stderr，不會干擾 MCP 通訊。

## 擴展

### 支援其他 PHP CMS

架構設計可替換 API Client：

```
src/client/
├── wp-api-client.ts        # WordPress（預設）
├── laravel-api-client.ts   # Laravel（可擴展）
└── drupal-api-client.ts    # Drupal（可擴展）
```

修改 `src/index.ts` 中的 client 初始化即可切換。

### 加入新 Tool

1. 在 `src/tools/` 建立新檔案
2. 實作 `registerXxxTools(server, client)` 函式
3. 在 `src/tools/index.ts` 的 `registerAllTools` 中加入
