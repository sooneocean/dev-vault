## 二、溝通時間表與 SLA

### 編輯 → 寫手（反饋物）

| 時間點 | 編輯提供 | 格式 | 寫手確認時間 |

| 場景 | 溝通方式 | 回應時間 |

## 三、決策優先級（衝突時用）

```
決策優先級（從高到低）：

1️⃣  【結構不能動】
    - 7 個 H2 的框架不變
    - TL;DR + FAQ 必須有
    - H2-5 或 H2-6 融入台灣內容（位置可調但必須有）

    理由：這是整個專案的脊骨。結構變了，風格也會受影響。

2️⃣  【精神內核不能動】
    - 福音樂必須被呈現為「身體與精神同步的修行」
    - Mahalia Jackson / Sister Rosetta Tharpe 的對比必須呈現出「聖潔 vs 自由」
    - 台灣融合必須展示「靈性相通」而非「文化對比」

    理由：這是 YOLO LAB 這篇文章的靈魂。失去了，文章就失去了差異化。

3️⃣  【風格可微調】
    - YOLO LAB 的「暴力親切」是原則，但實現方式可以多元
    - 寫手的個人聲音可以保留（只要符合品牌調性）
    - 某些段落的節奏可以微調（只要符合整體邏輯）

    理由：編輯在這層有妥協空間。好的風格是寫手個人特色 + 品牌調性的融合。

4️⃣  【細節可調整】
    - 字數 ±200 字可接受
    - 引述的形式可改（關鍵是內容不變）
    - 小標題的措辭可改（關鍵是邏輯不變）

    理由：細節不影響核心。只要大方向正確，細節的優化是自然的。

決策樹：
寫手提出異議
  ↓
  是否涉及「結構」？
  ├─ 是 → 編輯堅守（除非寫手有非常好的替代方案）
  └─ 否 → 是否涉及「精神內核」？
         ├─ 是 → 編輯堅守（同上）
         └─ 否 → 是否涉及「風格」？
                ├─ 是 → 協商，尋求融合（70% 品牌 + 30% 寫手聲音）
                └─ 否 → 編輯承認寫手想法更好，採納
```

## 五、常見修改場景 & 應對

### 場景 1：寫手說「精神內核太難，我改不了」

**症狀**：
- 文章讀起來像音樂歷史教科書
- 沒有「身體與精神同步」的感受
- 過度學術化或過度說教

**編輯回應**：
1. 指出具體段落：「H2-5 的這段讀起來像教科書……」
2. 提供修改方向：「比起解釋『黑人女性如何被邊緣化』，應該寫『在唱詩班裡，黑人女性第一次感受到自己的聲音有力量』」
3. 提供參考：提示寫手看 Unit 1-3 的相應段落，如何呈現精神體驗
4. 給寫手時間：如果問題嚴重，允許寫手要求多 1-2 小時的重寫時間

**最壞情況**：編輯自己補寫 2-3 段，與寫手協商融合

---

### 場景 2：寫手抗議「台灣融合太生硬，我不想加」

**症狀**：
- 寫手認為台灣案例「破壞了文章的整體性」
- 想要刪除台灣段落

**編輯回應**：
1. 確認原因：問寫手為什麼認為生硬（是邏輯不通？還是風格不搭？）
2. 如果是邏輯問題：編輯提供不同的台灣案例或不同的融合角度
3. 如果是風格問題：協商修改台灣段落的寫法，使其更自然
4. 最後的妥協：允許寫手將台灣融合從「20% 全篇分散」改為「10% 集中在一段」，但不能完全刪除

**絕不妥協**：台灣融合是品牌要求（客戶發佈在台灣媒體），不能刪除

---

### 場景 3：寫手提出「我想改大綱結構」

**症狀**：
- 寫手認為「7 個 H2 太多」或「某些 H2 應該合併」
- 提出完全不同的結構

**編輯回應**：
1. **聽寫手的想法**：問為什麼認為現有結構不好
2. **評估提案的合理性**：
   - 新結構是否符合「開篇 → 人物 → 技術 → 社會 → 後代」的邏輯？
   - 是否保留了「台灣融合點」？
   - 是否仍能呈現精神內核？
3. **決策**：
   - 如果新結構更好 → 採納（優先級：邏輯 > 原大綱）
   - 如果新結構和原大綱一樣好 → 堅守原大綱（避免變數）
   - 如果新結構更差 → 拒絕，解釋原大綱為什麼是最優

**關鍵**：大綱是合約一部分，改動需要雙方協商確認

---

### 場景 4：寫手超期提交

**症狀**：
- 04-15 沒交稿
- 寫手說「再給我 12 小時」

**編輯決策樹**：
```
超期多少？
├─ ≤ 6 小時（<04-15 晚間 + 6h） → 接受，但強調「最多再寬鬆 6 小時」
├─ 6-12 小時 → 接受，但這樣 04-16 的反饋會延到 04-17 上午
│                         → 04-17 的修改稿截止改為 04-17 23:00
├─ > 12 小時 → 原則上不接受，除非有正當理由
│             （生病、家庭急事等）
│             → 若接受，整個時間表順延
│             → 發佈日期 04-24 能否保住取決於後續進度

最壞情況：如果 04-17 還沒交初稿，啟動 Plan B
├─ Plan B 選項 1：編輯團隊自己寫（時間會很緊）
├─ Plan B 選項 2：邀請備選寫手（如果有的話）
├─ Plan B 選項 3：延期發佈到 05-01
```

---

### 場景 5：寫手說「Sister Rosetta Tharpe 的酷儿身份我不知道怎麼寫」

**症狀**：
- 寫手迴避這個話題
- 或寫得太直白/不敏感

**編輯回應**：
1. **主動協助**：編輯提供以下參考資料：
   - Sister Rosetta Tharpe 的傳記段落（專注於「靈性自由 vs 社會規範」）
   - 1950s 美國黑人教會對 LGBTQ+ 的態度（歷史背景）
   - 精神自由與性別認同的關係論述

2. **修改方向**：
   - 不是「作為酷儿，她被教會反對」（受害敘事）
   - 而是「她用樂器、節奏、身體的自由來實踐靈性，這挑戰了教會對『正統』福音樂的定義」

3. **給寫手示範**：
   提供 1-2 個範本段落，讓寫手參考「敏感但有力」的寫法

## 六、決策快速查表

### 我是編輯，我該……

| 情況 | 決策 | 行動 |
|------|------|------|
| 寫手提出「改大綱」 | 評估新大綱的合理性 | 更好 → 採納；一樣好 → 堅守；更差 → 拒絕 |
| 寫手提出「刪台灣部分」 | 原則上不同意 | 協商修改台灣段落的寫法使其更自然 |
| 寫手超期但理由充分 | 個案評估 | 接受並順延時間表，但最後截止日期（發佈）不動 |
| 寫手初稿精神內核有問題 | 重點反饋 + 提供修改方向 | 提出具體問題、提供參考案例、允許重寫時間 |
| 寫手初稿風格不符 | 協商修改 | 解釋 YOLO LAB 風格是什麼、舉例、給修改方向 |
| 寫手初稿結構偏離 | 要求改回原大綱 | 解釋原大綱的邏輯、顯示新結構哪裡有問題 |

### 我是寫手，我該……

| 問題 | 做法 | 不要做 |
|------|------|--------|
| 對 Brief 有疑問 | 立刻問編輯 | 自己猜編輯的意思然後偏離方向 |
| 覺得大綱不適合 | 提早溝通（04-10 前） | 寫到一半才說想改結構 |
| 精神內核寫不出來 | 問編輯要參考資料或範本 | 硬撐或跳過這個部分 |
| 台灣融合不知道怎麼寫 | 要求編輯提供台灣案例清單 | 硬加一段沒邏輯的台灣話題 |
| 超期沒法按時交 | 提前 48 小時告知編輯 | 到時間再說「我要延期」|
| 初稿被編輯否定 | 接受反饋、詢問如何改善 | 為自己辯護或拒絕修改 |
| 修改意見有疑問 | 電話或 Slack 及時澄清 | 自己猜編輯的意思，亂改一通 |

## 七、風險預警

### 黃色預警（需注意）

```
⚠️  寫手初稿提交時間接近 04-15 18:00
    → 編輯反饋會延遲到 04-16 下午
    → 寫手修改時間被擠壓
    → 04-17 可能出現品質問題

⚠️  寫手初稿字數大幅超標（>3,400 字）
    → 修改難度上升
    → 需要大幅刪除，風險是損傷精神內核
    → 建議提前介入，指導寫手刪減

⚠️  寫手初稿精神內核呈現不足
    → 第一輪反饋后的修改會很困難
    → 可能需要重寫 2-3 段
    → 時間會變得非常緊張

⚠️  台灣融合段落生硬
    → 可能需要多輪修改
    → 或編輯主動介入改寫
    → 協調成本上升
```

### 紅色預警（停止發佈）

```
🛑  04-17 還沒交修改稿
    → 已無法按 04-20 定稿
    → 啟動 Plan B：延期發佈 或 編輯團隊自己寫

🛑  初稿精神內核完全缺失（不是不足，是根本沒有）
    → 寫手能力與需求不匹配
    → 考慮終止合作，邀請備選寫手
    → 時間代價：2-3 天重寫

🛑  寫手拒絕修改關鍵部分（大綱 / 台灣融合）
    → 無法談判
    → 終止合作，啟動 Plan B

🛑  04-20 還未定稿
    → 發佈日期 04-24 無法保障
    → 必須延期發佈
    → 或發佈不完整版本（不建議）
```

【第一優先】精神內核

【第二優先】台灣融合

換個角度：不是「比較台灣和美國」，而是「在台灣，福音樂精神也活著」。

我們會提供 2-3 個台灣案例清單，你挑最有感覺的一個深化。

現稿 3,280 字，超目標的 80 字。可以删除冗餘的歷史背景段落。

下一步：
