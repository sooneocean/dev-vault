## P1 High Priority Issues

These should be fixed. High-impact defects likely to be hit in production.

### Batch Processing Issues
- **L177 (adversarial):** Exception aggregation — zip() mismatch between crop_indices and results when batch partially fails. No length validation.
  - **Fix:** Assert `len(crop_indices) == len(results)` or ensure inpaint_batch always returns same-length list (use None for failed crops).

### Timeout & Resource Handling
- **L255 (reliability):** ComfyUI health check timeout hardcoded to 5s — fails on high-latency networks or slow model loading.
  - **Fix:** Add `config.comfyui_timeout` (default 5s, configurable).
- **L97 (adversarial):** Timeout handling mixes asyncio and aiohttp semantics; doesn't catch all timeout exceptions.
  - **Fix:** Wrap async calls in `asyncio.timeout()` context manager (Python 3.11+); catch broad Exception and log type.

### Validation Gaps
- **L164 (correctness):** Mask dimensions hardcoded; should be read from actual frame dimensions.
- **L164 (adversarial):** No validation that extracted frames match assumed 1920×1080.
  - **Fix:** Read sample frame, assert dimensions, pass to mask_loader.
- **L362 (correctness):** crop_regions KeyError if frame was skipped in preprocessing.
  - **Fix:** Check `if frame_idx in self.crop_regions` before accessing; skip frame in stitch if metadata missing.
- **L296 (adversarial):** Mask file copy assumes static mask; no validation for dynamic BBOX/POINTS types.
  - **Fix:** Check mask type; if dynamic, generate per-frame masks; if static, copy once.

### State & Cleanup
- **L409 (correctness):** cleanup() missing inpainted_dir — orphaned temp files accumulate.
  - **Fix:** Add `Path(self.config.output_dir) / "inpainted"` to dirs_to_clean list.
- **L414 (adversarial):** Cleanup partial failure — if one rmtree fails, others silently skip. State corrupted.
  - **Fix:** Two-phase cleanup: validate all removable, then delete atomically. Or rename to .deleted and defer.

### Coordinate Math
- **L209 (correctness):** Off-by-one in crop clipping when context extends beyond frame edge.
- **L232 (adversarial):** _region_from_mask() doesn't clamp context bounds to frame dimensions.
  - **Fix:** Add `max(0, ...)` and `min(frame_w, ...)` to context_x/context_w.
- **L103 (correctness):** unpad operation unvalidated — if padding > crop size, produces empty/invalid array.
  - **Fix:** Validate y_start < y_end and x_start < x_end before slicing.

### Type & Configuration
- **All files (kieran-python):** Deprecated type hints using `Dict`, `List` instead of `dict`, `list` (PEP 585).
  - **Fix:** Replace `Dict[int, Path]` → `dict[int, Path]`, etc.
- **L54 (kieran-python):** Missing type annotation on `_region_from_mask(mask)` parameter.
  - **Fix:** Add `: np.ndarray` type hint.
- **L160 (kieran-python):** encode_from_config() ignores config.output_fps, always uses 30.0.
  - **Fix:** Use `config.output_fps` in subprocess call.

### Performance (Gated)
- **L154 (performance):** Feather mask nested pixel loop — O(H×W) Python loop vs vectorized NumPy.
  - **Fix:** Replace with numpy broadcasting; ~1000x faster.
- **L191 (performance):** Unnecessary uint8→float32 full-frame copy in stitch_back.
  - **Fix:** Convert only blended region, keep original in uint8.
- **L158 (performance):** Preprocessing loop fully sequential; could parallelize with ThreadPoolExecutor.
