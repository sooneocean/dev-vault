# agent-memory MCP 预设启用 — CLAUDE.md 需要禁用说明

## Problem

Journal 2026-04-06 Friction: "agent-memory 預設啟用 — pipeline 預設開啟 agent-memory MCP，需手動停用"

在 Unit 6 工作流中发现，Claude Code pipeline 默认启用了 agent-memory MCP 服务器，但项目已在 `settings.json` 中禁用：

```json
"enabledPlugins": {
  "claude-mem@thedotmack": false,  // explicitly disabled
  ...
}
```

问题是开发者不知道为什么禁用，下次升级或重新配置时可能重新启用导致冲突。

## Suggested Change

在 CLAUDE.md 的 MCP Servers 表格中添加说明：

```markdown
| **agent-memory** (disabled) | ❌ | Conflicts with obsidian-agent; use clausidian for vault management instead |
```

或在表格后添加脚注：

```markdown
**Note**: `claude-mem@thedotmack` is explicitly disabled because project uses `obsidian-agent` CLI for vault management. Enabling both can cause index conflicts.
```

## Expected Effect

- 开发者（包括 Agent）知道为什么禁用 agent-memory
- 减少误操作重新启用的风险
- 其他贡献者理解项目的 vault 管理方案

## Application Record

Applied 2026-04-12: Added note to CLAUDE.md MCP Servers table explaining that agent-memory is disabled to avoid conflicts with obsidian-agent vault management.
