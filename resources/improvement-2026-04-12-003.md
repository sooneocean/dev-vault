# Vault project 笔记维护指南 — 添加批量归档和 summary 补填流程

## Problem

Journal 2026-04-06 Friction: "vault project 筆記雜亂 — 29 個 active projects，多數 stale，缺少 summary/updated"

Vault 中有 29 个 `projects/` 笔记，但：
- 多数标记为 `status: active` 但实际已放弃
- 缺少 `summary` 字段（导致 review 和 list 命令输出不清晰）
- 缺少 `updated` 日期，无法判断是否过期
- 没有清晰的归档流程

这导致：
- 列出 active projects 时信息不足
- 无法通过 `obsidian-agent stale` 自动检测过期项目
- 手动翻阅所有 project 文件寻找可归档的项目

## Suggested Change

在 CONVENTIONS.md 的 Project Schema 或 Vault Maintenance 段落中添加：

### 1. Project metadata 强制字段

```markdown
**Required fields for projects:**
- `title`: Project name
- `status`: active | paused | archived | completed
- `summary`: 1-2 sentence description (required for active projects)
- `updated`: Date of last substantive update (YYYY-MM-DD)
- `domain`: Project category (from DOMAINS list)
```

### 2. 维护流程

```markdown
## Vault Maintenance (Monthly)

Run quarterly cleanup to keep vault healthy:

\`\`\`bash
# Find stale projects (no update in 90 days)
obsidian-agent stale --path projects/ --days 90

# Archive a project
obsidian-agent archive "projects/project-name"

# Bulk update summaries
obsidian-agent patch "projects/X" --frontmatter summary "Brief description"
\`\`\`

**When to archive:**
- No updates for 90+ days
- Status changed to `completed` or `paused`
- Project goals no longer relevant

**Before archiving:**
- Update `summary` if empty
- Set `status: archived`
- Run `obsidian-agent sync` to rebuild indices
```

### 3. 在 CLAUDE.md 中添加快速参考

```markdown
**Active Projects Checklist:**
- [ ] All active projects have `summary` field
- [ ] Run `obsidian-agent stale` monthly
- [ ] Archive completed/paused projects
```

## Expected Effect

- Project 笔记维护有清晰的流程
- `obsidian-agent stale` 和 review 命令输出更有用
- 减少每周审视 vault 时的手动翻阅工作
- 新 agent 知道何时以及如何维护项目笔记
