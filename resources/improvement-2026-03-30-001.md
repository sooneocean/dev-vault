# AGENT.md 缺少新 schema 欄位與完整 CLI 命令

## Problem

Plan 004 完成後，CONVENTIONS.md 新增了 `subtype`、`maturity`、`domain`、`relation_map` 四個 schema 欄位及其 controlled vocabulary。但 AGENT.md 的 "Rules for Manual Edits" 段落仍只提 "Include complete frontmatter"，沒有列出這些新欄位。

同時，AGENT.md Quick Start 的 CLI 範例缺少 `health`、`graph`、`patch`、`update`、`archive` 等常用命令，agent 可能不知道這些能力存在而回退到手動檔案編輯。

Journal 2026-03-30 Friction section 記錄了「遷移必須用 direct file editing」——部分原因就是 agent 不知道 CLI 有 `patch` 命令。

## Suggested Change

1. 在 AGENT.md "Rules for Manual Edits" 第 2 點後加入：

```markdown
3. **New schema fields**: `subtype` (resource only), `maturity` (seed/growing/mature), `domain` (controlled vocab — see CONVENTIONS.md), `relation_map` (typed relations)
```

2. 在 Quick Start CLI 區塊補充缺少的常用命令：

```bash
# Maintenance & discovery
obsidian-agent health                    # vault health score
obsidian-agent stale                     # find stale notes
obsidian-agent cluster                   # topic clustering
obsidian-agent digest                    # daily digest
obsidian-agent suggest                   # smart action suggestions

# Edit existing notes
obsidian-agent patch "note" --heading "TODO" --append "- [ ] New task"
obsidian-agent update "note" --status active --summary "Updated"
```

## Expected Effect

Agent 在手動編輯筆記時會知道必須填寫 subtype/maturity/domain 欄位，減少遷移後仍產出不完整 frontmatter 的機率。也會優先使用 CLI patch/update 而非 direct file editing。

## Application Record

Applied 2026-03-29: Added new schema fields (subtype/maturity/domain/relation_map) to AGENT.md "Rules for Manual Edits" section. Added discovery & analysis CLI commands (stale, cluster, digest, thread, suggest, context) to AGENT.md Quick Start.
