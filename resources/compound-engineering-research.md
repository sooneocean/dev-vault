# Compound Engineering 研究

## 核心問題

如何讓每次工程工作累積為下次的加速器，而非增加技術債？

## 核心原則

- 80% 規劃與審查、20% 執行
- 每個 cycle 都產出可複用的知識
- 品質高 → 未來修改容易 → 正向循環

## 工作流循環

```
Brainstorm → Plan → Work → Review → Compound → Repeat
```

| 階段 | 目的 | 複利機制 |
|------|------|---------|
| Brainstorm | 釐清需求 | 需求文件可重用 |
| Plan | 拆解實作 | 計畫模板累積 |
| Work | 執行 | worktree 隔離風險 |
| Review | 多 agent 審查 | 審查標準持續提升 |
| Compound | 記錄學習 | 知識直接餵回下一輪 |

## 與其他三學科的關係

- **Prompt Eng.** — Compound 的 brainstorm/plan 本質上是 prompt 設計
- **Context Eng.** — 知識文件化讓 context 更精準
- **Harness Eng.** — Plugin/agent 架構是 compound 工作流的基礎設施
