# 開源專案品質標準

## 重點

### 上架前必備清單（Must Have）

#### 文件
- [ ] **README.md** — 一句話說明、安裝步驟、使用範例、截圖/GIF
- [ ] **LICENSE** — 建議 MIT（與現有專案一致）
- [ ] **CHANGELOG.md** — 版本歷程紀錄

#### Repo 設定
- [ ] **Description** — 一行描述（出現在 GitHub 搜尋結果）
- [ ] **Topics** — 至少 5 個標籤（提高曝光度）
- [ ] **About 區塊** — 填寫完整

#### 程式碼品質
- [ ] **測試** — 至少有基本測試，CI 會跑
- [ ] **Linter** — ruff（Python）/ prettier + eslint（TS/JS）/ clippy（Rust）
- [ ] **無 Secrets** — .env 不入版控，有 .env.example

#### CI/CD
- [ ] **GitHub Actions** — 至少有測試自動化
- [ ] **Badge** — README 上顯示測試狀態、版本、授權

### 加分項（Nice to Have）

- [ ] **CONTRIBUTING.md** — 貢獻指南
- [ ] **Issue 模板** — Bug Report / Feature Request
- [ ] **PR 模板** — 標準化 PR 描述
- [ ] **程式碼文件** — API 文件、架構圖
- [ ] **Demo / 截圖** — README 中附上使用畫面
- [ ] **Release 自動化** — 用 GitHub Actions 自動建 Release

### 各語言專屬要求

#### Python
- [ ] `pyproject.toml` 完整（name、version、description、dependencies）
- [ ] `python -m build` 能成功打包
- [ ] 型別提示（type hints）至少在公開 API 上

#### TypeScript / Node
- [ ] `package.json` 完整（name、version、description、keywords、license）
- [ ] `tsconfig.json` 嚴格模式
- [ ] `npm run build` 能成功

#### Rust
- [ ] `Cargo.toml` 完整（name、version、description、license）
- [ ] `cargo clippy` 無警告
- [ ] `cargo doc` 能產生文件
