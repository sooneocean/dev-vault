# Windows POSIX 路径转换 — 自动修复 Claude Code commands 的路径格式

## Problem

Journal 2026-04-06 Friction: "POSIX 路徑 — Windows 環境下 Claude Code commands 的路徑格式不一致，需手動修正"

在 Windows 上运行 Claude Code，某些 commands 和工具返回 Windows 路径格式（`C:\Users\User\...`），但大部分脚本期望 Git Bash 格式（`/c/Users/User/...`）。这导致：
- 脚本中手动转换路径
- 路径变量混用导致 tool calls 失败
- 需要多次调整命令才能成功

## Suggested Change

在 `settings.json` 的 `PostToolUse` hooks 中添加一个新 hook，自动将所有 Windows 路径转换为 POSIX 格式：

```json
{
  "matcher": "Bash|Write|Edit|Read|Glob|Grep",
  "hooks": [
    {
      "type": "command",
      "command": "bash -c 'input=$(cat); echo \"$input\" | python3 -c \"import sys, json, re; data=json.load(sys.stdin); result=re.sub(r\\\"([A-Z]):\\\\\\\\([^\\\"]*)\\\", lambda m: \\\"/\\\"+m.group(1).lower()+\\\"/\\\"+m.group(2).replace(\\\"\\\\\\\\\\\", \\\"/\\\"), json.dumps(data)); print(result)\"'",
      "timeout": 3,
      "statusMessage": "Converting Windows paths to POSIX...",
      "async": false
    }
  ]
}
```

**Alternative simpler approach** (如果上面的 JSON 太复杂)：
在 `.bashrc` 或 PowerShell profile 中设置别名或函数，确保所有脚本输入的路径都是 POSIX 格式。

## Expected Effect

- Tool outputs（Glob, Read 等）自动返回 POSIX 路径
- 脚本执行时不再需要手动转换路径
- 减少"path not found"错误和重试
