# CONVENTIONS.md CLI 範例過時 — 缺少 v0.7.0 編輯與維護命令

## Problem

CONVENTIONS.md 的 "Using the CLI (recommended)" 段落只列出 7 個命令（journal, note, capture, search, list, review, sync），但 obsidian-agent v0.7.0 新增了大量命令。

特別是 `patch`（追加內容到指定 heading）和 `update`（更新 frontmatter）是避免 direct file editing 的關鍵命令，但完全沒出現在 CONVENTIONS.md。

Journal 2026-03-30 記錄的「遷移必須用 direct file editing」和「updateNote() 只能更新已存在的 key」兩個摩擦點，部分可用 `patch` 命令緩解。CLAUDE.md 已在 improvement-2026-03-29-001 更新了 CLI Quick Reference（加入 stale/cluster/digest/thread/suggest/context），但 CONVENTIONS.md 作為 agent 的首要參考文件卻被遺漏了。

## Suggested Change

替換 CONVENTIONS.md 中 "Using the CLI (recommended)" 的程式碼區塊：

```bash
obsidian-agent journal              # Create/open today's journal
obsidian-agent note "Title" type    # Create a note (auto-links related notes)
obsidian-agent capture "idea"       # Quick idea capture
obsidian-agent search "keyword"     # Search notes
obsidian-agent list [type]          # List notes
obsidian-agent review               # Generate weekly review
obsidian-agent sync                 # Rebuild indices

# Edit existing notes (prefer over direct file editing)
obsidian-agent patch "note" --heading "Section" --append "content"
obsidian-agent update "note" --status active --summary "Updated"
obsidian-agent archive "old-note"

# Discovery & maintenance
obsidian-agent health               # Vault health score
obsidian-agent stale                # Find stale notes
obsidian-agent suggest              # Smart action suggestions
obsidian-agent cluster              # Topic clustering
obsidian-agent context "note"       # Get note context with neighbors
```

## Expected Effect

Agent 讀 CONVENTIONS.md 時就能發現 `patch` 和 `update` 命令，優先用 CLI 而非 direct file editing。減少 frontmatter 損壞和 index 不同步的風險。

## Application Record

Applied 2026-03-29: Expanded CONVENTIONS.md "Using the CLI" section from 7 commands to full grouped reference (create/read, search/discover, edit, tags, reviews, discovery/analysis, maintenance) — 24 commands total.
