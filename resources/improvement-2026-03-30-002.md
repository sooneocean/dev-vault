# OA_TIMEZONE 未設定導致跨午夜 journal 日期錯誤

## Problem

Journal 2026-03-30 Friction 記錄：「`obsidian-agent journal` 用 UTC 時區判斷日期，跨午夜後仍產出前一天的 journal」。

AGENT.md 文件中記載 `OA_TIMEZONE` 環境變數可設定時區，但目前系統並未在任何地方設定此變數。而 `scripts/session-stop-wrapper.sh` 使用 `date +"%Y-%m-%d"` 也受系統時區影響。

對於 UTC+8 的使用者，每天 16:00-24:00（UTC 08:00-16:00）之間 journal 日期正確，但 00:00-08:00（UTC 前一天 16:00-24:00）期間會寫入前一天的 journal。

## Suggested Change

兩個互補的修正：

### 1. 設定 OA_TIMEZONE 環境變數

在 PowerShell profile 或 `.bashrc` 加入：

```bash
export OA_TIMEZONE="Asia/Taipei"
```

### 2. 在 session-stop-wrapper.sh 明確指定時區

```diff
- TODAY=$(date +"%Y-%m-%d")
+ TODAY=$(TZ="Asia/Taipei" date +"%Y-%m-%d")
```

```diff
- TS=$(date +"%H:%M:%S")
+ TS=$(TZ="Asia/Taipei" date +"%H:%M:%S")
```

### 3. 在 settings.json Stop hook 設定 env（可選）

如果 obsidian-agent CLI 不讀取 shell profile，在 hook command 前加：

```json
"command": "OA_TIMEZONE=Asia/Taipei bash \"/c/DEX_data/Claude Code DEV/scripts/session-stop-wrapper.sh\""
```

## Expected Effect

午夜到早上 8 點之間的 session 也能正確寫入當天的 journal，消除每天早上要手動修正 journal 日期的摩擦。
