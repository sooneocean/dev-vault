# Writing Methodology Framework

## The Idea

### Compound Engineering Writing Framework (CEWF)\n\n1. **Capture (Idea)**: use obsidian-agent capture to record sparks.\n2. **Research (Resource/Research)**: gather data, screenshots, and logs using obsidian-agent research.\n3. **Develop (Project)**: build scripts (like wpcom-batch-updater.py) to automate tasks.\n4. **Write (Resource/Article)**: draft in Obsidian with specific frontmatter (wordpress_id, publish_status).\n5. **Batch (Script)**: push subtype: article notes to external platforms (WordPress, etc.) using the script.

## Why

### Principles\n\n- **Dev-driven content**: Writing is based on research and code logs, ensuring technical depth.\n- **Obsidian as single source of truth (SSOT)**: Both development and writing happen in the same vault.\n- **Automation-ready**: All articles are structured and ready for batch processing.
