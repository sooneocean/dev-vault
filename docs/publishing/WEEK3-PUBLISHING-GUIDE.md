## 🚀 自動化發佈方法

### 方法 1: 使用 Python 腳本 (推薦)

```bash
# 步驟 1: 設置 WordPress.com API 令牌
export WP_TOKEN="your_wordpress_com_token_here"

# 步驟 2: 運行發佈腳本
python3 scripts/batch-create-week3-posts.py
```

**需要條件**:
- WordPress.com API 令牌 (從 [WordPress.com 設置](https://wordpress.com/me/security) 生成)
- Python 3.6+
- urllib (內置)
- json (內置)

### 方法 2: 手動發佈 (WordPress 後台)

如果自動化發佈失敗，可以手動創建：

1. 登錄 WordPress.com → yololab.net
2. 導覽到 「文章」→「新增」
3. 複製文章內容 (去掉 frontmatter)
4. 設置發佈時間為對應的 UTC+8 時間
5. 設置分類和標籤
6. 設置 URL slug
7. 點擊「排程發佈」

**優點**: 可以檢查內容格式和圖片
**缺點**: 耗時 (10 篇文章需要 20-30 分鐘)

---

### 方法 3: WP-CLI (如果網站是自架 WordPress)

如果網站不是 WordPress.com，而是自架 WordPress：

```bash
# 安裝 WP-CLI
curl -O https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar
chmod +x wp-cli.phar
sudo mv wp-cli.phar /usr/local/bin/wp

# 發佈文章
wp post create article-06-kpop-neogen.md \
  --post_type=post \
  --post_status=future \
  --post_date="2026-04-15 18:00:00" \
  --post_category=Music
```

## 📝 獲取 WordPress.com API 令牌

1. 訪問 https://wordpress.com/me/security
2. 點擊「應用程式密碼」或「個人令牌」
3. 創建新令牌，選擇「文章:寫」權限
4. 複製令牌值並保存到環境變數

```bash
export WP_TOKEN="paste_your_token_here"
echo $WP_TOKEN  # 驗證設置
```

## ❌ 故障排除

### 問題 1: WP_TOKEN 無效
**症狀**: `HTTP Error 401: Unauthorized`
**解決**: 重新生成令牌，確保權限包含「文章:寫」

### 問題 2: 分類不存在
**症狀**: `Error: Category not found`
**解決**: 在 WordPress 後台手動創建分類 (Music, Film, Tech)

### 問題 3: 發佈時間格式錯誤
**症狀**: `Invalid date format`
**解決**: 確保時間格式為 ISO 8601 (2026-04-15T18:00:00+08:00)

### 問題 4: 標籤過多
**症狀**: `Tag limit exceeded`
**解決**: 減少每篇文章的標籤數量 (建議不超過 5 個)
