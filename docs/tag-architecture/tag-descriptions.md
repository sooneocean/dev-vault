## Content Structure & Usage

### Tag Page Template (Block Editor)

Each super-tag page should include:

1. **Header Block:** Tag name + description (from above)
2. **Featured Content Block:** 3-5 recent high-quality articles
3. **Related Tags Block:** Links to related super-tags or sub-clusters
4. **Statistics Block (Future):** Post count, trending topics, audience metrics
5. **Call-to-Action Block:** "Subscribe to {Cluster} Updates"

### Metadata (Jetpack SEO)

Each super-tag page should have:
- **SEO Title (55-60 chars):** `{Cluster} News & Analysis | YOLO LAB`
- **Meta Description (155-160 chars):** First 2 sentences from description above
- **Robots:** `index, follow`
- **Canonical:** `https://yololab.net/tag/{slug}/`

### OpenGraph Tags

- **og:title:** `{Cluster} Stories | YOLO LAB`
- **og:description:** Snippet of cluster description
- **og:image:** YOLO LAB logo or cluster-specific imagery
- **og:url:** `https://yololab.net/tag/{slug}/`

## Editorial Guidelines

### When Creating Tag Descriptions

1. **Use Active Voice:** "YOLO LAB examines..." not "This tag covers..."
2. **Lead with Value:** Why should readers care about this cluster?
3. **Distinguish from Silos:** Explain how this differs from similar topics
4. **Include Specific Examples:** Reference actual coverage areas
5. **Signal Authority:** Use confident, knowledgeable language
6. **Stay Under 200 Words:** Conciseness respects reader time

### When Linking Between Tags

- Link to related super-tags in the "Related Topics" section
- Use anchor text that previews the linked content
- Avoid orphaning low-traffic clusters

### When Assigning Articles to Tags

- Every article should have ≥1 super-tag assignment
- Prefer 1-2 super-tags per article (avoid over-tagging)
- Use sub-cluster anchors (#events, #personalities) for rich context
- Wildcard tags remain acceptable for high-specificity content
