# YOLO LAB 編輯標準指南

## 核心理念

**YOLO LAB = 撕開平庸的暴力介入**

我們的音樂內容不是被動的「評論」，而是主動的「社會診斷」。每一篇文章都應該：
- 展現強烈個人立場（毒舌但有據）
- 連接技術分析 + 社會意義
- 用當代語言拆解深度內涵
- 讓讀者有「被看見」的感覺

---

## I. 標題與 SEO 層

### 標題公式
```
[角度] + [主人公/作品] + [核心insight] + 修飾詞 | YOLO LAB
```

**範例**：
- 「【樂評】比宿醉更噁心的是瞎咖！Yohee又熙聯手屁孩、Barry Chen《如果我是個玩咖》撕開「免費WiFi男」的噁心假面」
- 「Central Cee 2026 新碟解析：從聲學工藝看見 UK Drill 的暗黑巔峰」
- 「從瘋子到王者：Kanye West《Bully》全解析，揭開 2026 嘻哈圈最強救贖」

**規範**：
- 字數：55-65 字（SEO 黃金範圍）
- 包含主人公名字 + 作品名稱
- 提示「角度」（樂評、解析、拆解、揭開等）
- 帶有「個性化」的修飾詞（暴力、暗黑、毒舌等）
- 中英文混用自然，避免生硬

### Meta Description
```
[鉤子句。]深入解析 [主人公] [作品名]，[核心洞察]。樂評評分 X.X/10。
```

**規範**：
- 字數：155-160 字
- 開頭帶「鉤子句」（2-3 詞，製造好奇心）
- 中段說明「是什麼」
- 結尾帶「評分」（信號完整性）

**範例**：
- 「Yohee又熙強勢回歸。《如果我是個玩咖》用社會學隱喻精準擊碎中央空調假象，屁孩Barry Chen聯手暴力美學重擊渣男標本。新世代情感演算法指南。樂評評分 8.7/10。」

### Jetpack Publicize Message
```
【評分 X.X/10】[Title]：[核心insight] #tag1 #tag2
```

**規範**：
- 限制 280 字以內（Twitter/X 相容）
- 評分醒目化（【評分】格式）
- 用 3 個 hashtag 提高社群觸及
- 簡潔有力，帶有號召性

## II. 內容結構層

### 標準架構（必須遵循）

```
1. H1 開篇鉤子（1 段落）
   └─ 直接對話讀者，製造緊張感或共鳴

2. [TL;DR] 重點快讀（Bullet points，3-4 條）
   └─ 核心論點 + 關鍵數據/時間點 + 方法論 + 實用價值

3. 主體段落（4-6 個 H2 主題）
   ├─ 第一個 H2：深層事實 / 背景 / 靈感來源
   ├─ 第二個 H2：技術分析（音樂、製作、結構）
   ├─ 第三個 H2：社會意義 / 隱喻 / 文化位置
   ├─ 第四個 H2：具體解讀（歌詞、音色、編排）
   ├─ 第五個 H2：YOLO LAB 總編點評 / 終極評價
   └─ 第六個 H2（可選）：額外維度（市場、歷史、預測）

4. 一句話總結（可選，但強烈建議）
   └─ 讓讀者帶走一個記憶點

5. 🎵 延伸閱讀區塊（3-5 篇相關文章）
   └─ 內部連結 + descriptive anchor text + 簡要說明
```

### 字數目標
- **總字數**：2,500-3,500 字
- **TL;DR**：100-150 字
- **主體段落**：2,200-3,200 字
- **延伸閱讀**：150-200 字

## III. 語調與聲音層

### YOLO LAB 音樂評論的「聲音」特徵

| 向度 | 標準 | 禁止 |
|------|------|------|
| **主觀性** | 強烈個人立場，用「我」、「我們」 | 「據報導」「有人說」這種躲避 |
| **詞彙風格** | 流行文化用語、俚語、時代感 | 陳腔濫調、老氣、教科書式 |
| **節奏感** | 短句 + 長句交錯，造成快慢節奏 | 等長句子，讀起來單調 |
| **目標讀者** | 直接對話讀者（「你」、「我們」） | 第三人稱、客觀評述 |
| **毒舌度** | 尖銳但有據，有幽默感 | 單純罵人、無建設性批評 |
| **引用風格** | 歌詞引用 + 解讀，生動舉例 | 堆砌樂理術語，無人文關懷 |

### 具體範例

**好的 YOLO LAB 語調**：
- 「在這個交友軟體滑到手抽筋、感情像速食一樣廉價的年代，我們聽過太多哭得死去活來的失戀情歌。」
- 「聲學工藝與頻率顛覆」（小標題要有衝擊力）
- 「戴上耳機的瞬間，這17分鐘的西倫敦街頭暗黑低頻將徹底重塑你的聽覺認知。」
- 「只有親自站在這陣音浪中心，你才會明白西倫敦的街頭霸主如何用頻率統治世界。」

**避免的 YOLO LAB 語調**：
- 「該專輯由 X 製作人負責製作…」（太官方）
- 「根據音樂學分析，此作品運用了…」（太學術）
- 「許多評論家認為…」（躲避個人立場）

## IV. 技術分析層（音樂評論專用）

### 必須涵蓋的三個維度

#### 1. 聲學工藝 / 製作層面
- **要素**：錄音質感、母帶處理、頻率下潛、聲場設計
- **呈現**：不用樂理術語堆砌，而是用**聽覺感受**+ **製作意圖**來解讀
- **範例**：「混音台上的每一推桿都在為那股冷血、精準的UK Drill骨架服務」

#### 2. 社會學 / 文化意義
- **要素**：時代背景、身份政治、社會現象、代際差異
- **呈現**：用**隱喻**連結音樂到社會現實，讓讀者產生共鳴
- **範例**：「感情像Wi-Fi，誰都能連」（隱喻渣男行為）

#### 3. 技藝與靈魂
- **要素**：歌手/樂手的個人簽名風格、創新點、進化軌跡
- **呈現**：追蹤藝人的演變，指出這作品在其生涯中的位置
- **範例**：「Central Cee將視角拉回Shepherd's Bush的公車站牌」

### 禁止項目
- ❌ 「根據 Spotify 數據」這種無關內容
- ❌ 純技術術語堆砌（BPM、Hz、dB 等）
- ❌ 離題的八卦或緋聞
- ❌ 抄襲官方新聞稿

## V. 台灣在地化層（黑人音樂人系列專用）

### 80-20 原則（嚴格遵循）
- **80%**：國際音樂史基礎 + 深度分析（保持權威性）
- **20%**：台灣本地案例 + 台灣讀者關心的維度

### 自然融合位置（不生硬）

| 位置 | 用途 | 範例 |
|------|------|------|
| **主體段落中段** | 介紹台灣音樂人受影響的案例 | 「正如伍佰在 1990 年代融合布魯斯與台灣搖滾…」 |
| **技術分析段** | 台灣本地的技術/製作案例 | 「台灣製作人在這個頻率領域的探索…」 |
| **社會意義段** | 台灣讀者能直接共鳴的社會現象 | 「就像台灣年輕人面臨的身份焦慮…」 |
| **延伸閱讀區塊** | 連結到台灣相關藝人評論 | 「見《伍佰評論》或《陶喆靈魂音樂繼承者》」 |

### 禁止項目
- ❌ 「台灣的音樂產業如何…」（統計式本地化）
- ❌ 生硬的「台灣人也喜歡…」類語句
- ❌ 為了本地化而扭曲歷史事實

## VI. 內部連結策略層

### 延伸閱讀區塊格式

```markdown
## 🎵 延伸閱讀：[主題]

- [Descriptive Anchor Text](URL) — [一句簡要說明，點出相關性]
- [Descriptive Anchor Text](URL) — [一句簡要說明]
- [最多 5 篇，最少 3 篇]
```

### 連結選擇邏輯

**優先順序**：
1. **簇內連結**（同系列文章）：Jazz → Blues → Soul/R&B → Gospel → Hip-Hop 的縱向連結
2. **橫向連結**（相同藝人）：Blues 文章連結到「伍佰評論」（受影響案例）
3. **主題連結**（相同風格/年代）：Gospel 文章連結到「台灣教會音樂」相關內容

**描述文案規範**：
- 簡潔（不超過 15 字）
- 指出「為什麼相關」，而非單純「點擊看」
- 帶有「延伸維度」的提示（「古往今來」、「音樂到社會」、「技術到精神」）

**範例**：
```
- [Central Cee 新碟解析：UK Drill 的暗黑巔峰](URL) — 同代西方低頻美學的極限挑戰
- [伍佰評論：80 年代台灣搖滾與藍調融合](URL) — 看台灣如何消化黑人音樂基因
- [Kanye West《Bully》全解析](URL) — 跨時代的聽覺工程與文化致敬
```

## VII. 品質驗收清單（寫手交稿前）

### 內容質量 ✓
- [ ] 標題 55-65 字，帶有個性化修飾詞
- [ ] Meta 描述 155-160 字，包含評分
- [ ] TL;DR 3-4 條 bullet，涵蓋核心論點、數據、方法、實用性
- [ ] 主體 4-6 個 H2，依序涵蓋：背景、技術、社會、具體解讀、編輯點評
- [ ] 字數 2,500-3,500 字（含延伸閱讀）
- [ ] 一句話總結（記憶點明確）

### 語調檢查 ✓
- [ ] 用第一人稱（「我」「我們」），直接對話讀者
- [ ] 包含 2-3 個流行文化或時代用語
- [ ] 至少 1 段用隱喻連結音樂與社會
- [ ] 毒舌但有據，幽默感明顯
- [ ] 沒有「據報導」「有人說」類躲避句子

### 技術分析 ✓
- [ ] 涵蓋聲學工藝/製作層面（用聽覺感受而非樂理術語）
- [ ] 涵蓋社會學/文化意義（用隱喻連結）
- [ ] 涵蓋藝人個人簽名風格與進化軌跡
- [ ] 無樂理術語堆砌（BPM、Hz、dB 只在必要時出現）
- [ ] 無無關資訊（八卦、官方數據、Spotify統計）

### 台灣在地化 ✓
- [ ] 80% 國際基礎 + 20% 台灣案例（嚴格比例）
- [ ] 台灣融合點自然出現，無生硬感
- [ ] 包含台灣藝人案例（伍佰、陶喆、Yohee、Central Cee 等）
- [ ] 沒有統計式本地化或為了本地化而扭曲歷史

### 內部連結 ✓
- [ ] 延伸閱讀區塊有 3-5 篇相關文章
- [ ] 連結文案簡潔（不超過 15 字）
- [ ] 指出相關性，而非「點擊看」
- [ ] 優先連結簇內文章（Jazz → Blues → Soul 等順序）

### 元數據 ✓
- [ ] Featured Image（專輯封面或藝人照片，高清）
- [ ] Categories（音樂評論分類）
- [ ] Tags（3-5 個，包含藝人名、風格、年代）
- [ ] Jetpack Publicize Message（280 字以內，帶評分）

## VIII. 招募寫手框架

### 寫手必須具備的能力
1. **音樂素養**：能聽懂製作層面 + 能寫出聽覺感受
2. **社會洞察**：能連結音樂到當代社會現象
3. **中文表達**：繁體中文自然流利，能用網路用語而不顯生硬
4. **台灣視角**：能自然融入台灣本地案例，不強行本地化

### 寫手準備物資
- 本編輯指南（這份文檔）
- 參考範本：Yohee 評論 + Central Cee 評論（作為聲音模板）
- 黑人音樂人系列框架文檔（每篇文章的具體大綱）

### 寫手交稿流程
1. **草稿提交** → 編輯初審（內容質量 + 語調）
2. **修改迴圈** → 最多 2 輪修改
3. **終審發布** → 驗收清單 100% 通過 → 排版發布

## IX. 常見錯誤 & 修正

| 錯誤 | 原因 | 修正方式 |
|------|------|---------|
| 標題太長（>70字） | 貪心想塞入所有資訊 | 刪除修飾詞，只保留「角度 + 主人公 + 核心insight」 |
| 全用樂理術語 | 寫手來自音樂學背景 | 改寫成「聽覺感受」，用比喻而非術語 |
| 台灣案例太多（>30%） | 為了SEO本地化 | 嚴格遵循 80-20 原則，刪除非必要本地化 |
| 沒有社會連結 | 純技術分析 | 加入「這代表什麼」的思考，用隱喻連結 |
| 延伸閱讀無故事 | 只是堆砌連結 | 每條連結加「為什麼相關」的一句說明 |
| 語調太官方 | 抄襲官方新聞稿 | 全部改寫成第一人稱 + 直接對話讀者 |
