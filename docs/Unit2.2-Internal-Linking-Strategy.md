## 第 2 步：內部連結規劃與錨文本編寫

### 連結規劃原則

**基本方針**：
1. **支柱頁反向連結**：每個 TOP 文章都應連回其分類的支柱頁（≥1）
2. **同分類相關連結**：同分類的其他 TOP 文章互相連結（≥2-3）
3. **主題相關連結**：特定主題（如 Yohee、AI 工具）的文章互相連結（≥1-2）
4. **發現更多**：季末或分類末頁添加「發現更多」連結到支柱頁

### 連結類型與位置

#### 類型 1：支柱頁反向連結（Priority: P0）

**位置**：文章 H2 末段或結論段落

**錨文本模式**：`[查看完整 {{category}} 分析]` 或 `[{{關鍵詞}} 完整指南]`

**範例**：

```markdown
### Yohee 樂評內容中的反向連結

文章結尾段落：
"...透過嘻哈語言重新定義了當代渣男人設。想深入理解更多樂評分析？
[查看 YOLO LAB 樂評評測完整指南](https://yololab.net/archives/music-pillar)"

✅ 錨文本：「查看 YOLO LAB 樂評評測完整指南」
✅ 位置：結論段落（自然、非强制）
✅ 相關性：100% （同分類支柱頁）
```

#### 類型 2：同分類相關連結（Priority: P1）

**位置**：文章正文 H2/H3 中間（自然融入）

**錨文本模式**：`{{相關藝人/主題}} [完整分析|樂評|評測]`

**範例**：

```markdown
### Kanye West 樂評中的同分類連結

文本：
"...Kanye 的製作理念深受 Dr. Dre 影響，後者的製作哲學可見於
[Jacob Collier 的編曲分析](https://yololab.net/archives/jacob-collier)中。"

✅ 錨文本：「Jacob Collier 的編曲分析」
✅ 位置：正文中間（自然提及）
✅ 相關性：Kanye → Jacob Collier（製作理念相通）
```

#### 類型 3：主題相關連結（Priority: P2）

**位置**：側欄、相關文章區、或主題引導

**範例**：

```markdown
### AI IDE 文章的主題相關連結

文本：
"...使用 GitHub Copilot X，一個基於 GPT-4 的編程助手。
更多 [AI 工具對比](https://yololab.net/archives/llm-comparison) 見相關文章。"

✅ 錨文本：「AI 工具對比」
✅ 相關性：AI IDE → AI 工具總覽（主題相通）
```

## 第 4 步：批量實施策略

### 方案 A：手動編輯（適合小規模 ≤20 篇）

```
步驟：
1. 進入 WordPress 編輯器 → 編輯文章
2. 在規劃的位置（正文、結論段落）找到對應句子
3. 選擇要連結的詞語 → 按 Ctrl+K（或點工具欄鏈接按鈕）
4. 輸入目標文章 URL
5. 設置為「在新頁籤打開」（可選）
6. 更新文章
```

**優點**：精確控制、可優化錨文本
**缺點**：耗時 30-40 分鐘 per 文章

### 方案 B：使用 WordPress CLI 腳本（適合大規模 >50 篇）

```bash
# 偽代碼示例（需適配您的環境）
wp post list --post_type=post --posts_per_page=50 --format=csv > top50.csv

# 使用 Python/Node.js 腳本批量添加內部連結
# 需要解析內容、找到錨點、插入 <a> 標籤、更新文章
```

**優點**：高效批量處理
**缺點**：需要技術能力、風險高

### 方案 C：混合方案（推薦）

```
第 1 天：手動編輯 TOP 10 最高優先級文章（支柱頁反向連結）
第 2-3 天：編輯次級文章的同分類連結（腳本輔助）
第 4 天：驗證所有連結有效性
```

**耗時估計**：12-16 小時

## 第 5 步：連結有效性驗證

### 驗證清單

```
□ 每篇 TOP 文章確實有 ≥ 5 個內部連結
□ 所有連結都是「活連結」（無 404 或損壞連結）
□ 錨文本多樣化（不是全部 「點這裡」 或 「了解更多」）
□ 連結相關性 ≥ 80%（大多數連結確實相關）
□ 同分類互連率 ≥ 60%
□ 沒有過度優化（無垃圾連結或關鍵詞堆砌）
```

### 驗證工具

1. **Screaming Frog**（SEO 工具）：批量檢查破損連結
2. **WordPress 插件**：「Broken Link Checker」
3. **手動檢查**：隨機抽樣 10-20 篇，點擊驗證

## 第 6 步：互連率計算與報告

### 互連率公式

```
同分類互連率 = （相同分類之間的連結對數） / （該分類文章的總數） × 100%

範例（音樂分類）：
- 音樂 TOP 5 文章：Yohee, Kanye, Jacob, Central Cee, Sarah Chen
- Yohee 連結到其他 4 篇：4 個互連
- Kanye 連結到其他文章：3-4 個
- Jacob 連結到其他文章：2-3 個
- 合計：≥12 個互連
- 互連率：12 / (5*5) = 48%（達到目標 ≥60% 需擴展到 15-20 篇）
```
