## 技术架构

### API 调用流程

```
[WordPress.com API: posts.list]
  └─ 获取 50 篇文章（分页）
     └─ [WordPress.com API: posts.get] × 50
        └─ 获取每篇标题+摘要
           └─ [Anthropic API: messages]
              └─ 生成 SEO 标题+描述
                 └─ [WordPress.com API: posts.update]
                    └─ 更新元数据
                       └─ [ProgressTracker]
                          └─ 记录进度/错误
```

### 错误处理流程

```
[API 调用]
  ├─ 成功 (200) → addSuccess()
  ├─ 失败 (非 200)
  │  ├─ 重试次数 < 3 → 重试（指数退避）
  │  └─ 重试次数 >= 3 → addFailed()
  └─ 网络错误 → addFailed()
```

## 脚本参数配置

### 可调整参数

在 `seo-batch-automation.js` 中修改 `CONFIG` 对象：

```javascript
const CONFIG = {
  SITE_ID: '133512998',           // WordPress.com 站点 ID
  ANTHROPIC_MODEL: 'claude-opus-4-6-20250805',  // AI 模型
  BATCH_DELAY: 750,               // 批次延迟 (毫秒)
  TIMEOUT_PER_POST: 30000,        // 超时 (毫秒)
  RETRIES: 3,                     // 重试次数
  BATCH_SIZE: 1,                  // 批次大小（保持 1）
};
```

### 建议配置

**生产环境** (默认):
- `BATCH_DELAY`: 750ms
- `TIMEOUT_PER_POST`: 30s
- `RETRIES`: 3

**加速模式** (谨慎使用):
- `BATCH_DELAY`: 500ms
- `TIMEOUT_PER_POST`: 20s
- `RETRIES`: 2

## 容错和恢复策略

### 自动恢复机制

| 错误类型 | 处理方式 | 继续处理 |
|---------|---------|---------|
| 网络超时 | 重试 3 次 | ✅ |
| API 限流 | 等待后重试 | ✅ |
| SEO 生成失败 | 记录 ID | ✅ |
| 元数据更新失败 | 记录 ID | ✅ |
| 凭证错误 (401) | 终止 | ❌ |

### 中断恢复步骤

1. **停止脚本** (Ctrl+C)
2. **查看进度**
   ```bash
   cat seo-batch-logs/progress.json
   ```
3. **确认失败的文章**
   ```bash
   cat seo-batch-logs/failed-posts.json
   ```
4. **重新运行脚本**
   ```bash
   node scripts/seo-batch-automation.js
   ```
5. **脚本自动跳过已处理的文章**
